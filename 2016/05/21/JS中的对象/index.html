JavaScript中的对象

作为一个前端新人，一直对面向对象认识的不是很清楚，大家都在说面向对象的特征:封装,继承,多态。知道了概念，看看自己的代码却依然还是流水账式。不能在实践中应用学到的知识，那是无法进步的。

基本概念

面向对象术语

摘自w3school

对象

ECMA-262 把对象（object）定义为“属性的无序集合，每个属性存放一个原始值、对象或函数”。严格来说，这意味着对象是无特定顺序的值的数组。
尽管 ECMAScript 如此定义对象，但它更通用的定义是基于代码的名词（人、地点或事物）的表示。

类

每个对象都由类定义，可以把类看做对象的配方。类不仅要定义对象的接口（interface）（开发者访问的属性和方法），还要定义对象的内部工作（使属性和方法发挥作用的代码）。编译器和解释程序都根据类的说明构建对象。

实例

程序使用类创建对象时，生成的对象叫作类的实例（instance）。对类生成的对象的个数的唯一限制来自于运行代码的机器的物理内存。每个实例的行为相同，但实例处理一组独立的数据。由类创建对象实例的过程叫做实例化（instantiation）。
在前面的章节我们提到过，ECMAScript 并没有正式的类。相反，ECMA-262 把对象定义描述为对象的配方。这是 ECMAScript 逻辑上的一种折中方案，因为对象定义实际上是对象自身。即使类并不真正存在，我们也把对象定义叫做类，因为大多数开发者对此术语更熟悉，而且从功能上说，两者是等价的。

对象的构造

看javascript高级程序设计这本书的时候，知道了对象有很多种构造方式。下面将详细的列出各种构造方式示例，以及为何使用的原因

工厂模式

function createCar(sColor,iDoors,iMpg) {
  var oTempCar = new Object;
  oTempCar.color = sColor;
  oTempCar.doors = iDoors;
  oTempCar.mpg = iMpg;
  oTempCar.showColor = function() {
    alert(this.color);
  };
  return oTempCar;
}

var oCar1 = createCar();
var oCar2 = createCar();

这是新手最常用的方式，通过每次调用函数创建一个空对象再为其赋值从而返回不同的对象。这种方法一直被诟病，一方面是由于这看起来并不像其他语言一样用new实例化，显得不那么正规，另一方面我们可以看到，每次创建对象都在对象中创建了一个showColor函数(造成了内存的浪费,影响性能)，但我们的目的其实是多个实例共享同一个函数。

所以这种方法被改造为：

function showColor() {
  alert(this.color);
};

function createCar(sColor,iDoors,iMpg) {
  var oTempCar = new Object;
  oTempCar.color = sColor;
  oTempCar.doors = iDoors;
  oTempCar.mpg = iMpg;
  oTempCar.showColor = showColor
  return oTempCar;
}

var oCar1 = createCar();
var oCar2 = createCar();

由于函数是引用类型，只保存了对象的“址”（指针），所以每次创建对象时候不会多次创建showColor函数。同时保证每个实例都可以调用showColor方法。但是这种方法依然没有解决语义化的问题（并没有使用new关键字）所以又有开发者定义了构造函数方式

构造函数方式

构造函数模式构造相同对象的代码如下：

function Car(sColor,iDoors,iMpg) {
  this.color = sColor;
  this.doors = iDoors;
  this.mpg = iMpg;
  this.showColor = function() {
    alert(this.color);
  };
}

var oCar1 = new Car("red",4,23);
var oCar2 = new Car("blue",3,25);

这种方式使用了new关键字，使代码语义化。

而且在构造函数内没有创建对象，而是使用 this 关键字。使用 new 运算符构造函数时，在执行第一行代码前先创建一个对象，只有用 this 才能访问该对象。然后可以直接赋予 this 属性，默认情况下是构造函数的返回值（不必使用 return 运算符）（加重字体说明了new运算符到底做了什么）。

这样做同样创建了多个showColor函数实例，你当然也可以在函数外部定义showColor再添加引用，但是这样就又不在语义化了。

原型方式

每个对象都有一个prototype属性，这个属性可以看做是创建这个对象的原型。比如说数组的prototype是Array。

原型方式创建对象代码示例如下：

function Car() {
}

Car.prototype.color = "blue";
Car.prototype.doors = 4;
Car.prototype.mpg = 25;
Car.prototype.showColor = function() {
  alert(this.color);
};

var oCar1 = new Car();
var oCar2 = new Car();

这种方式创建出的对象都具有相同的属性，同时并没有示例化多个showColor函数。而且有属性instanceof 指向他们的创建者。

比如由于typeOf不能判断一个数组，那么可以使用instanceOf来判断

var a = [1,2,3]
a instanceOf Array == true

但是原型方式也不尽人意，因为这种方式创建出的对象的属性都是相同的，我们必须实例化一个对象以后再一个个改掉他们的属性。保存对象的引用是正确的，但是属性并不需要一样。

混合的构造函数/原型方式

我们既然已经见识了以上各种方式，那么我们可以将他们的优点结合：

function Car(sColor,iDoors,iMpg){
    this.color = sColor;
    this.doors = iDoors;
      this.mpg = iMpg;
      this.drivers = new Array("Mike","John");
}

Car.prototype.showColor = function(){
    alert(this.color)
}

var car1 = new Car("red",4,23);
var car2 = new Car("blue",4,25)

car1.driver.push("Stack")

alert(car1.drivers);    //输出 "Mike,John,Bill"
alert(car2.drivers);    //输出 "Mike,John"

car1.showColor()         //输出 "red"
car2.showColor()         //输出 "blue"

这种方法的showColor方法只创建了一个函数，并且使用了new操作符，又有各子的属性，不会多个实例共享相同的属性，使用了原型方式，所以能用instanceOf判断创建者。

但是有些开发者仍然认为这样不够完美，代码哲学中将对象对象看做是现实个体的反映，世界是由对象间的互相作用产生的。（你不能先创建半个个体，再创建另一半）。

动态原型方法

动态原型方法的基本想法与混合的构造函数/原型方式相同，即在构造函数内定义非函数属性，而函数属性则利用原型属性定义。唯一的区别是赋予对象方法的位置。下面是用动态原型方法重写的 Car 类：

function Car(sColor,iDoors,iMpg) {
  this.color = sColor;
  this.doors = iDoors;
  this.mpg = iMpg;
  this.drivers = new Array("Mike","John");

  if (typeof Car._initialized == "undefined") {
    Car.prototype.showColor = function() {
      alert(this.color);
    };

    Car._initialized = true;
  }
}

该方法使用标志（_initialized）来判断是否已给原型赋予了任何方法，该方法只创建并赋值一次。

总结

这些东西是js中需要理解的东西，理解的越深，越能在代码中得心应手的应用，同时是我自己学习的一些要点的记录，也希望能够帮到和我一样新的同学。

我个人认为混合模式能更多的使用，因为_initialized这种属性在代码中没有实际的作用。
